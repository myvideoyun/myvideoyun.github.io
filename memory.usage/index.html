Collect Memory Usage of C/C++ executables on Ubuntu

Concepts

  • Heap:
  • Stack
  • RSS: Resident Set Size: the size of memory which are in RAM. It does not include memory which is swapped out.
  • VSZ: Virtual memory size;

Tools

Top

The top program provides a dynamic real-time view of a running system. It can display system summary information as well as a list of processes or threads currently being managed by the Linux kernel.

It can display following fields: - %CPU: Cpu usage - %MEM: Memory Usage (RES), share of available phisical memory, its non-swapped phisical memory a task is using. - %DATA: Data + Stack Size (KiB), Data Resident Set size or DRS - SHR: Shared Meomory Size: The amount of shared memory available to a task, not all of which is typically resident. It simply reflects memory that could be potentially shared with other processes. - SWAP: Swapped Size: The non-resident portion of a task’s address space. - VIRT: Virtual Memory Size, The total amount of virtual memory used by the task. It includes all code, data and shared libraries plus pages that have been swapped out and pages that have been mapped but not used. - nTH: Number of Threads: The number of threads associated with a process.

Time

/usr/bin/time, it will give the summary of the memory usage.
